# Branch `develop`

## Objetivo

A branch `develop` é a etapa de **homologação e consolidação** das mudanças vindas de `features`.

## Responsabilidades

- Receber PRs aprovados de `features`.
- Consolidar incrementos já validados no nível de feature.
- Preparar o pacote estável para promoção à `main`.

## Fluxo de trabalho

1. Receber mudanças de `features` via PR.
2. Revisar integração entre funcionalidades.
3. Validar estabilidade da entrega.
4. Abrir PR de `develop` para `main`.

## Regra do fluxo

`features` -> `develop` -> `main`
